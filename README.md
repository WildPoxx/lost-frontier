# OLF - OutsiderS Lost Frontier

Repositorio matriz do projeto **OutsiderS - Lost Frontier**, reunindo canone, design de jogo, documentacao de producao e desenvolvimento do modulo Foundry VTT.

## Escopo

- RPG de campanha e cenario para Savage Worlds / SWADE.
- Material autoral de lore, faccoes, personagens, bestiario, itens, handouts e regras de mesa.
- Desenvolvimento do modulo Foundry VTT `outsiders-lost-frontier`.
- Integracoes auxiliares para automacao, macros, compendios, assets e documentacao.

## Alvos tecnicos

- Foundry VTT v13 Stable build 351.
- SWADE 5.2.6.
- Forien's Quest Log como integracao recomendada, nao como dependencia obrigatoria de todo o projeto.

## Estrutura principal

- `01. Colonias e Locais/` - locais, colonias e geografias jogaveis.
- `02. Espécies/` - ancestralidades, povos e especies.
- `03. Aventuras/` - aventuras, atos e estruturas de sessao.
- `04. Personagens/` - PCs, NPCs e registros de personagem.
- `05. Facções/` - faccoes, organizacoes e conflitos.
- `06. Bestiário/` - criaturas, estatisticas e pacotes de bestiario.
- `07. Lore/` - canone, cosmologia e documentos de mundo.
- `08. Handouts/` - documentos para jogadores e material diegetico.
- `09. Veiculos e Tecnologia/` - veiculos, tecnologia e matriz tecnica.
- `10. Itens/` - equipamentos, armas, gear e itens autorais.
- `11. New Edges and Abillities/` - edges, habilidades e opcoes SWADE.
- `12. IA e Foundry Tools/` - ferramentas de IA, desenvolvimento Foundry, modulos, macros e pacotes.
- `13. Audio/` - materiais de audio.
- `14. Maps e Battle Maps/` - mapas e battlemaps.
- `15. House Rules and Rules/` - regras de mesa e referencias mecanicas.

## Relacao com MasterQuest

O repositório **MasterQuest** deve ser tratado como projeto associado: ele pode fornecer arquitetura, automacoes ou componentes de Foundry, mas este repositório permanece como fonte matriz do canone e da producao de **OutsiderS - Lost Frontier**.

## Politica inicial de Git

Este repositorio nasce privado e seletivo. Materiais de terceiros, instaladores, ZIPs, PDFs, runtimes locais, assets pesados e exports temporarios devem ficar fora do Git comum. Assets curados podem ser adicionados depois com Git LFS ou excecoes explicitas.

## Colaboracao IA

Codex e Claude devem usar `AI_COLLABORATION.md` como protocolo compartilhado de trabalho.
